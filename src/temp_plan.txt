Raise an explicit InvalidArchiveStructure exception if no target file matches.Step 3: Row Parsing & Routing RulesIterate over the rows of each discovered CSV file using the following precise data mapping maps:File A: TV Episode Tracking (records-v2.csv)For each row, read and extract these fields:show_name (String) -> The name of the series.season_number (Integer) -> Season index.episode_number (Integer) -> Episode index.watched_at (String/Timestamp) or updated_at.progress (String/Percentage).Routing Logic:IF progress equals "100%" OR watched_at contains a valid datetime string:Route to the History Array.Normalize the timestamp to strict ISO 8601 UTC format (YYYY-MM-DDTHH:MM:SSZ).If the timestamp string is completely missing but marked 100%, fallback to setting the execution date/time as the watch date.ELSE IF row has is_watchlisted == true OR (watched_at is empty AND progress is < 100%):Route to the Watchlist Array.File B: Movie Tracking (records.csv)For each row, read and extract these fields:movie_name (String)release_date or year (Integer)type or action status markers.Routing Logic:If the row represents a completed view, map movie_name, year, and watched_at into the Movie History Array.If marked as planned/queued, push to the Movie Watchlist Array.Step 4: Data Deduplication & Data Integrity LogicBefore trying to sync, run a data cleanup engine over your loaded arrays:Unique Key Constraint: Create a composite lookup key for each item:For TV: UniqueKey = ShowName + "_" + SeasonNumber + "_" + EpisodeNumber + "_" + WatchedDateFor Movies: UniqueKey = MovieName + "_" + ReleaseYearLogic: Pass arrays through a filter to drop identical duplicates. If the exact same episode is marked watched on the exact same second, preserve only the first entry.Step 5: Entity Resolution System (Fuzzy Matching Engine)Because TV Time data uses raw strings and internal tracking IDs rather than universal IDs, your agent must implement a tiered resolution loop:Tier 1 (Hard IDs): If the row provides a direct imdb_id or tvdb_id, map this immediately to your output payload.Tier 2 (Text Match): Query your destination database or API using the raw string title combined with the release year filter.Tier 3 (Normalization Clean): If Tier 2 returns 0 matches, pass the title string through a regex filter to strip special characters, accents, and trailing metadata strings (e.g., convert "The Office (US)" to "The Office").
